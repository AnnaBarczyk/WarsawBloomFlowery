    7. A) Template Versions - top

    The template comes in four different versions: light, video, blue and dark.

    • Light Version - this is the basic version of the template. It has a header slider, active fade In animation with the dominant color: pink. Files related to it are index.html and style.css. Less command for compile css is "gulp less"
    • Video Version - this is the template with a video background instead of slider. Also the video is displayed for quote and contact sections. It has an intro text section on top of the video and active fade In animation with dominant color: pink. Files related to it are index-video.html and style-video.css. Less command for compile css is "gulp less-video"
    • Blue Version - this is the basic version with blue accents and no animation fade effect and it has a header slider. Files related to it are index-blue.html and style-blue.css. Less command for compile css is "gulp less-blue"
    • Dark Version - this is dark version of the template. It has header slider with intro text section, active fade In animation with dominant color: orange and dark background. Files related to it are index-dark.html and style-dark.css. Less command for compile css is "gulp less-dark"

    B) HTML Structure - top

    This theme is created as a one-page site. It has a beautiful, clean and airy design. The template was created using inovative and powerfull technology as HTML5/CSS3 based on bootstrap (css framework) which makes development and maintanance very easy and intuitive. It supports all types of devices (desktops, tablets and smartphones) with several optimizations for each size.

    The whole project is divided into big sections. Most of them have a name placed on the top-right corner as a headline.

    1. At the very top we have the "header" section which consists of navigation, slider/video-background and optionally "intro-text".

    • The navigation has a fixed position at the very top of the page that you can see even when you scroll down. It is a nice transition effect as well.
    • The slider consists of 4 slides with a background image attached to each of them in CSS. Please check 'carusel1' style. When you will replace the images just place them in the "img/carousel" folder and change the style in CSS file.
    • In case of video-background, the slider section is deleted.

    2. After "header" we have a big list of sections which are simple and self-explanatory. I will describe them in brief where necessary:

    • Latest Project - it's a nice section which could represent your latest project and "find more" opens a modal window (first portfolio)
    • Our Story - here we have a simple text section devided into two columns
    • Our Team - the place for a short description and presentation of your team
    • Quote - a quote arrea with nice parallax effect background
    • Services - it's a section with good structure and great fade in effect
    • Newsletter - a basic newsletter form with an integrated validation
    • Portfolio - it's the more complex section with a hover effect and a pop-up modal which opens on click. This modals can be found by their identifier ("portfolioModal1", "portfolioModal2", etc.). There are also filter options which will group the elements and filter them using animation
    • Partners - a very simple section for displaying multiple logos or images
    • Testimonials - consists of a carousel which displays 2 elements at once
    • Contact - it's a form based section with validations
    • Footer - contains the address and social media links with s nice hover effect

    Whole HTML is realized in bootstrap style which makes it flexible, responsive and very easy to manipulate.

    C) CSS Files and Structure - top

    I'm using twitter bootstrap css framework as a base. As a result we have 2 files (bootstrap.min.css and style.min.css)

    • bootstrap.min.css - is a base library file
    • style.min.css - is a minified version of style.css file where we have all our custom styles. CSS is optimized and minified with gulp tool, which here is used to optimize/clean/minify css. I will describe it in a separate section related to "gulp tool".

    D) JavaScript - top

    This template includes some Javascript scripts for some animation and form validations as well as a tool to send emails to server-side mail service script.

    1. jQuery (jQuery is a Javascript library that greatly reduces the amount of code that you must write)
    2. Bootstrap Javascript file as part of framework.
    3. JQuery Easing - plugin for smooth using of navigation.
    4. JQuery viewportchecker - plugin for fade in animation effect(In case you don't need animation you can delete it).
    5. JQuery bcSwipe - plugin for easy navigate throw testimonials and header slider
    6. JQuery imagesloaded - plugin for portfolio section
    7. JQuery masonry - plugin for portfolio section
    8. jqBootstrapValidation - A plugin for automating validation on Twitter Bootstrap formatted forms(subscribe and contact).
    9. main - theme related scripts.

    E) Gulp Tool - top

    Gulp is a tool used to make some repeatable tasks. To run gulp, first you need to have nodejs and npm installed and only 2 files: package.json and gulp.js
    In order to run this automated tasks first install all dependencies with "npm install" and after run gulp tasks ("gulp uncss", "gulp minify-css"), both in root of the project from command line. Please check "gulp" documentation for more details.
    Gulp is used for the following automated tasks:

    • less - compile less file and combine with variables and create final css file. There are custom comands for each version (light template: less, dark template: less-dark, video template: less-video, blue template: less-blue)
    • uncss - cleaning of unused css. It is set to default css of light version. In case you want to minify other version please change css file name in uncss section gulpfile.js
    • compatibility - makes sure that the theme will looks good on all suported browsers.
    • minify - prepare the css for production by compresing the file.

    F) Customization - top

    This template can be customized in the easiest way possible. Usually you need to search through the whole project in order to change a font, color, backgorund-image. For an easier customization we created a configurations file "/less/variables.less". This file contains only the options for fonts, colors and background images which are set in CSS. Here you can change all the colors/fonts/images according to your needs and requirements. All the values are self-explained and grouped in an intuitive way. If you want to change text and information, you can do this in the HTML file.

    Note: After each CSS variables update (in case you update just HTML changes should be visible) you need to run gulp commands (in project root) to make changes visible:

    • "gulp less" - compiles CSS changes and applies variables
    • "gulp minify-css" - creates production-ready versions of CSS. You have to run this command if you use the minified CSS version (style.min.css)

    Note: In case you don't want to use gulp just change the CSS file which is imported in HTML to style.css and make all the changes in this file.
